Release Notes for the
Microsoft® Windows® Code-Named "Longhorn" SDK

Welcome to the Developer Preview Edition of the "Longhorn" Software Development Kit (SDK). The "Longhorn" SDK contains documentation, samples, command-line compilers, and tools designed to help you develop applications and libraries that target the Microsoft® Windows® (code-named "Longhorn") operating system.

Please note that this release is for preview purposes only. The APIs, documentation, samples, and tools provided in this release are preliminary and subject to change.

Because this is a pre-release edition, many of the known issues are identified in this document. Please read the installation instructions and known issues before installing. See the "Longhorn" Developer Center and check the "Longhorn" Beta Newsgroups for updates to known issues.

For additional information about "Longhorn", see The MSDN online "Longhorn" Developer Center. The "Longhorn" Developer Center is focused on keeping developers up-to-date on how they can take advantage of "Longhorn's" rich new client features in their applications. Visit the "Longhorn" Developer Center often to see new articles, read monthly columns, participate in the "Longhorn" newsgroups, download sample code, and read the Editor's Web Log.

Instructions for Installing "Longhorn" and the "Longhorn" SDK

  1. From the "Longhorn" setup program, install the "Longhorn" operating system. 
  2. From the "Longhorn" setup program, install the "Longhorn" SDK.
  3. Follow the instructions in the "Longhorn" SDK Setup wizard, accepting the default options.
  4. In some cases you may need to restart your computer to have your environment variables recognized.

Known Issues

This release has the following known issues. For additional information about "Longhorn", see The MSDN online "Longhorn" Developer Center.


"Longhorn" SDK Issues

Tools: FxCop not included in this release of the "Longhorn" SDK

Visit the following location in order to download a version of FxCop that works with the operating system on this CD: http://www.gotdotnet.com/team/fxcop/FxCopInstall1.30_Whidbey.exe

Samples: The samples in the "Longhorn" SDK need to be built on the command line

This version of the "Longhorn" SDK is not fully compatible with previously released versions of Microsoft® Visual Studio® "Whidbey." As a result, the samples will not build using previous versions of Visual Studio. Some samples pages incorrectly provide instructions for building the samples in Visual Studio. Instead, use the command line tools provided in the "Longhorn" SDK to build the samples.

Sample: "WinFS" sample requires data before it can run

For the sample code associated with the "Longhorn" storage subsystem (code-named "WinFS") API topic "How to Modify an Item and Update Changes in a Store" to run properly, there must be at least one document saved in the store prior to running the sample code.  To work around this issue, copy a document to the "WinFS" store before running the sample code or first run the sample code found in the "How to Create an Item and Save It to a Store in 'WinFS'" topic, which creates a Document item and saves it to the "WinFS" store.

Sample:  "WinFS" sample with DeleteFolder method fails if embedding or holding relationships exist on the target folder

The DeleteFolder sample method defined in several examples and used in some "WinFS" topics deletes a specified folder. If you apply this method to delete a folder that is a target of an embedding or holding relationship, the method will fail.  To work around this issue, delete embedding and holding relationships targeting the folder if you are attempting to delete using this method.

Samples: "WinFS" API samples do not support localization in this release

The "WinFS" samples are not localized and might fail when using certain localized characters. Localization is not fully supported in this release.

Building C++ projects that include "XAML" files

To compile "Longhorn" C++ projects that include "XAML" files, you must first run %windir%\microsoft.net\windows\v6.0.4030\RegisterCppCodeProvider.js, which is installed by the "Longhorn" SDK.

Building managed C++ samples from the command line

To build a managed C++ sample, open the "Longhorn" SDK Build Environment Command Prompt window and go to the source directory of the sample (for example: cd "C:\Program Files\Microsoft Longhorn SDK\Samples\MySample"). At the command prompt, type VCBUILD /useenv and press Enter.

Sample:  "Longhorn" Graphics Model ImageData

The sample Using ImageData is unavailable for both C# and "XAML".

 Sample: "Create a Storage Under Another Storage in a Container"

 The sample "Create a Storage Under Another Storage in a Container" is unavailable for VB.NET.

Chinese characters display improperly in two topics: "'Avalon' Composite Fonts Overview" and "Create and Use a Composite Font."

Question marks appear in the place of Chinese characters within code snippets in two topics: "'Avalon' Composite Fonts Overview" and "Create and Use a Composite Font."  To work around this issue, if you have East Asian Language Support installed on your computer, you will see the correct characters if you download and view the sample, "Create a Composite Font."

To download the sample:

  1. In the "Longhorn" SDK, click the Samples tab.
  2. On the Samples page, find the "Longhorn" Graphics Model section.
  3. Within that section, click "Create a Composite Font."
  4. Follow the instructions to download and run the sample.
     

Presentation Subsystem (Code-Named "Avalon") Issues

Issues with using a DrawingBrush as a Resource

In this release, a DrawingBrush cannot be used as a resource without performing a workaround. An InvalidOperationException is thrown when attempting to use the resource for the first time in a rendering operation. The following is an example of markup that causes an InvalidOperationException to be thrown:

<DockPanel.Resources>

<DrawingBrush

 

  def:Name = "BRUSH_MASK_BLACK_GRIDMARGINS"

 

  ViewPort = "0,0,2,4"

  ViewPortUnits = "Absolute"

  TileMode="Tile"

  ViewBox="0,0,1,1">

  <DrawingBrush.Drawing>

    <Rectangle
       RectangleTop ="0"
       RectangleLeft ="0"
       RectangleWidth ="1"
       RectangleHeight=".5"
       Fill = "#000"/>

  </DrawingBrush.Drawing>

</DrawingBrush>

</DockPanel.Resources>

 

....
 

<Button Background="{BRUSH_MASK_BLACK_GRIDMARGINS}">Button</Button>

The workaround ensures that the first access to the resource occurs before the resource is used in a rendering operation.  For example, the previous example will work if you add the following code to your application's constructor or StartingUp event.

Brush b = (Brush) myDockPanel.Resources["BRUSH_MASK_BLACK_GRIDMARGINS"];

Setting Opacity property has no effect on ImageBrush and DrawingBrush

In this release, setting the Opacity of ImageBrush and DrawingBrush has no effect. The Opacity property does have an effect on SolidColorBrush, LinearGradientBrush, and RadialGradientBrush. There is no workaround at this time.
 

"Longhorn" Help Issues

System.Help.Canvas API calls are ignored

In this release, API calls to System.Help.Canvas are ignored. An instance of the Canvas object can be created, but API calls do not function. No errors or exceptions are returned. There is no workaround.

Microsoft Assistance Markup Language ("MAML") schema files

The "MAML" schema consists of a set of .xsd files that are used to create and validate "Longhorn" Help content. Maml.xsd is the root-level file that should be referenced for validation. The files can be found installed with the "Longhorn" SDK at: C:\Program Files\Microsoft Longhorn SDK\misc\MAML_Schema\.

"Longhorn" Help Build Schema Files

The "Longhorn" Help build schema files are used to create the various Help project files. The files can be found installed with the "Longhorn" SDK at: C:\Program Files\Microsoft Longhorn SDK\misc\Help_Build_Schemas\.

"Longhorn" Help Compiler

The "Longhorn" Help compiler is used to create a .HELP file from a set of loose Help project files. Two files are required to make the compiler work: HelpCompiler.exe, and WindowsBase.dll. Copy these files to a local directory and follow the instructions in the documentation to use the compiler. The compiler files can be found installed with the "Longhorn" SDK at: C:\Program Files\Microsoft Longhorn SDK\bin\HelpCompiler.exe, and C:\Program Files\Microsoft Longhorn SDK\bin\WindowsBase.dll.
 

"WinFS" Issues

"WinFS" API FindOne and FindOnly methods deprecated

The "WinFS" API FindOne and FindOnly methods are deprecated. Instead, use the FindAll method and limit the results with the OPath filter.
