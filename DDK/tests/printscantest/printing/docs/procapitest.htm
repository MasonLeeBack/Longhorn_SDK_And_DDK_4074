Print Processor API Test

Introduction

This test is designed to verify the system compatibility of the print processor.  It does not test whether the print processor works properly.

Scope

The Print Processor API test (ProcAPITest.exe) is a Microsoft® Windows® codename “Longhorn” test that is to be included in the Longhorn Driver Kit (LDK).

 

 

Global Information

Background Information

Primary Operating System

Microsoft Windows Longhorn

 

Supported Operating Systems

Microsoft Windows 2000, Microsoft Windows XP, and Microsoft Windows Server™ 2003.

 

Device\Driver Category

This test applies to the print processor category.

 

Related DDK Topics

DDK\Display and Print Devices\Design Guide\Printer Drivers and Spooler Components\Customizing Print Spooler Components\Writing a Print Processor

 
Friendly Name

Print Processor API Test

 

Internal Name

ProcAPITest.exe

 

Description

This test loads the default print processor of a specified printer into its memory space and verifies the existence of all required interfaces.

 

Theory of Operation

This console-based test loads the default print processor for a specified printer into its process. It then tries to obtain the memory addresses of the six required interfaces. If a required interface is not present, the test logs it as a failure and moves on to the next interface.

 

Installation

Installation Steps

Copy the binary to the local machine. The test requires WTTLOG.dll.

 

File Location

The file is located at <LDK dir>\tests\printscantest\printing\<Arch>\ProcAPITest.exe.

You should install the file to <LDK dir>\tests\printscantest\printing\<Arch>\ProcAPITest.exe.

 

Removal Steps

You can delete the files by using the LDK uninstall program. You might have to delete the log files manually.

 

Environment Requirements

Required Software

Windows XP, Windows Server 2003, Windows Longhorn, or later operating system, and a printer driver.

 

Diagnosability

Setup Verification

There is no setup verification because you only need to copy an executable file to your hard drive.

 

Test Log Diagnosis

When a specific Print Processor Interface is marked as failed in the log, it means that the test was unable to obtain the memory address of that interface and that the interface is missing or spelled incorrectly.

 

Test-Specific Information

General Test Information

Manual vs. Automated

This is an automated test.

 

Logo Test–Specific Information

Logo Testing Verification

The test returns a log with pass or fail.

 

Test Duration

The test requires one to two minutes.

 

Restart

This test does not require a restart.

 

Default Configuration

The default configuration for this test is a clean installation of Windows Longhorn and standard installation of a printer.

 

Command Line Arguments

The printer name, as displayed in the Printers and Faxes folder.

 

Troubleshooting

The most common problems for this test are: a misspelled printer name, no printer installed, or no WTTLOG.dll on the system.
