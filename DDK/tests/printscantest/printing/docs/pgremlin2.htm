Scope – Longhorn Test

Print Gremlin 2 (pGremlin2.exe) tests the basic rendering primitives of a printer driver by calling various GDI APIs to render content.

 

Global Information

Background Information

Primary Operating System

Microsoft® Windows® codename “Longhorn”

Supported Operating Systems

Microsoft Windows 2000, Microsoft Windows XP, Microsoft Windows Server™ 2003.

Device\Driver Category

GDI Drivers - print, and fax drivers.

Related DDK Topics

DDK\Display and Print Devices\Design Guide\Graphic Drivers

Friendly Name

Printer Driver Compatibility Test

Internal Name

pGremlin2.exe

Description

This test calls various GDI APIs to produce output to verify that the driver renders graphics properly.

Theory of Operation

This is a console based test that produces output that needs to be manually verified.

 

Installation

Installation Steps

Copy the binary to the local computer. The test requires WTTLOG.dll.

File Location

The location of the files to be installed for this test:

<LDK dir>\tests\printscantest\printing\pGremlin2\<Arch>\pGremlin2.exe

These files are installed to:

<LDK dir>\tests\printscantest\printing\pGremlin2\<Arch>\pGremlin2.exe

Removal Steps

The files are deleted by the Microsoft Windows codename “Longhorn” Driver Kit (LDK) uninstall program.  Users may need to delete the log files. Fonts that are installed by the test may be left on the system.

 

Environment Requirements

Required Hardware

None 

Required Software

Windows XP, Windows Server 2003, Windows Longhorn or later, and a printer driver

Diagnosability

Setup Verification

There is just a single .exe file that needs to be copied.

Test Log Diagnosis

The output needs to be manually verified.  Every other page after the first is a bitmap copy of what GDI rendered to a memory DC (that is, it represents that output that is produced by GDI alone).  The content produced by the driver should match the bitmap output, except for the exceptions noted below.  Further, the pages marked as Robustness do not need to be reviewed for accuracy; they are produced to make sure that the driver output will not cause printer issues.

The following are the exceptions mentioned above for when the output, that the driver produces, isn’t expected to match the GDI created bitmap:

1)       When the printer isn’t capable of rendering bitmap. For example, line printers or text only devices.

2)       The device font wheel will, for the GDI rendered bitmap case, either print using a font other than the device font or will be blank. In either case, the comparison between the device font wheel rendered by GDI and the driver/printer isn’t valid.

3)       For printer drivers whose language that they produce is vector based (such as PostScript, PCL XL, HPG/L), the TransparentBlt tests may show black lines.  This is caused by a problem in GDI for this specific case.
