SCSI Compliance Test

Introduction

This test verifies that a SCSI device supported by the operating system fully complies with the SCSI standards described in the SCSI-3 Primary Commands (SPC), SPC-2, and SCSI Block Commands (SBC) specifications (to obtain these specifications, see the T10 Technical Committee Web site at http://www.t10.org/). Ensuring that the SCSI device and the operating system maintain compliance with these standards makes for a more robust and reliable system.

The test uses SCSI pass-through requests (IOCTL_SCSI_PASS_THROUGH) to construct and send SCSI command descriptor blocks (CDB) to the device. The test evaluates the results of the commands to verify compliance.

Friendly Name

SCSI Compliance Test

Description

The SCSI Compliance test tool verifies that the commands used by a SCSI block device are compliant with the functional requirements of SPC-2 (SCSI Primary Commands – 2) and SBC (SCSI-3 Block Commands) industry standards. A block device is a device that is capable of containing data stored in blocks that have a unique logical block address.

Theory of Operation

The test tool verifies that a command used by a SCSI block device is compliant with the requirements by checking the return value of the command that is sent to the device through the SCSI Pass Through interface (IOCTL_SCSI_PASS_THROUGH). First, the test creates a SCSI CDB (Command Descriptor Block) with the command that we want to test. Then, it sends the CDB to the device, gets the return value, and verifies the return value.

Installation

Installation Steps

Copy the following files to any folder in the test computer:

·         Scsicompliancetest.exe

·         WTTlog.dll

File Location

The location of the files to be installed for this test or code sample:

%SDXROOT%\testsrc\basetest\drivers\storage\compliance\objchk\i386

The user can install the test tool on any folder in the computer.

Removal Steps

The user needs to manually remove the Scsicompliancetest.exe, WTTlog.dll, and the generated test log files (in the same folder where the Scsicompliancetest.exe is installed) from the system.

Diagnosability

Setup Verification

The tool provides no verification that setup was completed correctly.

Debugging Activation and Output

The tester can activate the debugging option by specifying various verbose levels on the –v command-line option of the tool. The following example shows how to specify the highest verbose level for debugging purpose:

Scsicompliancetest.exe –v 2

Below is a list of verbose levels that the tester can use:

·         0 = Show the title of assertions and their results only.

·         1 = Show the details of assertions and their results (Default).

·         2 = Show all SCSI commands and data sent to the device, and their return values (sense information).

Note that running the tool with the highest verbose level may result in a log file that exceeds 200 MB.

After specifying this option, the tester will see extra debugging information in the test log (in the same folder where the tool is installed).

Test Log Diagnosis

The test identifies each test by the assertion number in the following format:

·         Major assertion number to specify the SCSI command to be tested.

·         Minor assertion number to specify the test case to be run on the SCSI command specified by the major assertion number.

For example, the following statement refers to the first test case of the SCSI command INQUIRY (Assertion 1).

ASSERTION 1 INQUIRY Command Test

Start: ASSERTION 1.1 INQUIRY Simple command test, TUID=(null)

Each assertion is followed by an information statement that specifies the section of the SPC-2 or SBC specification that the tester should refer to for more details on the functional requirement that is used to test the assertion.

Industry Specifications

T10 publishes the following industry specifications for SCSI storage interfaces. The tester needs to reference these documents to interpret the test logs.

·         SPC-2 (SCSI Primary Commands – 2) specification (available at http://www.t10.org/). 

·         SBC (SCSI Block Commands) specification (available at http://www.t10.org/).

Command Line Arguments

ScsiComplianceTest [-v level] [-full] [-hct] [-iscsi] [-mi device] [-alldevices] [-?] device

-v level              Specifies the verbosity level.

                        0 = List assertions tested and associated results.

                        1 = Show assertion details (Default).

                        2 = Show all SCSI commands and data that is sent to the device, and their return values (sense information).

-full                  Specifies that the destructive test should run. The destructive test writes random data on the device to be tested. Be sure that you have saved the data on the device to be tested.

-hct                  Runs as part of the HCT test pass.

-iscsi                Tests whether the commands of an iSCSI device are compliant to the iSCSI (instead of the SCSI) specification.

-mi device         Runs the multi-initiator tests on the specified device.

-alldevices        Runs the test on all SCSI block devices in a system.

-?                     Displays usage information.
