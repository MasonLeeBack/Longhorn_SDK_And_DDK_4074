Scope – Longhorn Test or LDK Sample

This document should be used for Display Conformance tests (DCT) (which are “Designed for Windows” Logo tests).

 

Global Information

Background Information

Primary Operating System

Designed to run on Microsoft® Windows® 98 Second Edition (SE) or later operating systems with Microsoft DirectX® version 7 or later.

 

Supported Operating Systems

Designed to run on Microsoft Windows 98 SE or later operating systems with Microsoft DirectX version 7 or later.

 

Device\Driver Category

Display Adapters

 

Related DDK Topics

These tests are designed to test the features documented in the Microsoft DirectX SDK (version 7.0 or later), and the Microsoft DirectX DDK (version 7.0 or later).

 

Friendly Name

This document is intended to cover all DCT applications in the M7 version of the Microsoft Windows codename “Longhorn” Driver Kit (LDK).  Collectively they are known as the D3D suite of DCT tests.

 

Description

These tests are designed to test the features documented in the DirectX SDK (version 7.0 or later) and the DirectX DDK (version 7.0 or later).  More information about each individual test can be found in chapter 5 of the WHQL Test Specification which can be found at http://www.microsoft.com/whdc/hwtest/pages/specs.mspx.

 

Theory of Operation

The specific details of what each test does and how it works can be found in chapter 5 of the WHQL Test Specification at http://www.microsoft.com/whdc/hwtest/pages/specs.mspx.

 

Environment Requirements

Required Hardware

The display adapter must be installed with the corresponding independent hardware vendor (IHV) driver.

Required Software

DirectX 7.0 or later must be present to successfully run the tests.  DirectX 9.0 is the preferred version since some tests will only work with DirectX 9.0.

 

Unsupported Hardware

All display hardware that supports DirectX 7.0 Display Driver Interface (DDI) or later is supported.

 

Tester Knowledge Requirements

The tester can run the test executable with /? or -? as a command-line option to get a detailed description of the available controls and input options.  There are many DCT tests, but an example of this would be:

Agp.exe /?

 

Diagnosability

Setup Verification

To a certain extent, if the proper files are not copied during setup, the job will fail.  The test will not run if DirectX version 7 or later is not present. DirectX 9.0 is the preferred version since some tests will only work with DirectX 9.0.

 

Debugging Activation and Output

Run the exe with the -? or /? switch to see how to control output:

<testname>.exe /? where <testname> is any of the DCT tests.

 

Test Log Diagnosis

The logs will contain information about each failure and there will be an associated command line in the log file that will help reproduce the error.

 

Debug Instructions

Once a failure occurs during the initial test, the test can be run again from a command line using the command line that is specified in the log file.  Then, the user will have more control of the test so they can step through each individual test case and then use other test features to gather more information.

 

 

Test-Specific Information

General Test Information

Manual vs. Automated

All of these tests are totally automated.

 

Program Affiliation

Logo testing for display adapters.

 

Logo Test Specific Information

Logo Program Requirements

There is a specific Logo Program Requirement tied to the DCT display tests.  Each test must be run in order to have a submission that is capable of receiving a “Designed for Windows” Logo.  The test detects the capabilities of the display adapter and determines which, if any, tests cases need to be run.

 

Logo Testing Verification

The log will summarize the number of passes and fails.  There must be zero failing test cases for a passing status.

 

Test Duration

The duration is heavily dependent on the hardware’s features.  The tests will detect the features available and turn off or turn on certain test cases based on available features.

 

Reboot

No reboot is required.

 

Default Configuration

Installed display driver with compatible hardware.

 

Interference

Screensavers must be disabled and power management must be turned off.

 

Command Line Arguments

Run the test with -? or /? for a complete explanation of the command-line parameters.

 

Troubleshooting

Always check log results.  Confirm that the test and support files are successfully copied to the client. If a job fails after a test is run, you should check to see if a log is present.  If a log is not present, run the test again from a command line. If a log is present after running the test outside of the job, then a problem with copying the log to the server exists, or there is a problem with how the log is parsed by the Windows LDK.  If there is a log with failures, review the log to determine the type of failures.
