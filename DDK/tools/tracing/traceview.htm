TraceView Utility

 

SUMMARY

TraceView is a tool which can be used to control and display Event Tracing For Windows (ETW) trace information. The goal of this tool is to make it easy and convenient for driver developers to use ETW in their drivers.  This document describes the current release of TraceView.  For more information, please see the help file (TraceView.chm) provided with this utility.

Current STATUS

The current version of TraceView is V1.9.1, which is the initial complete release of this tool.  TraceView is supported on both Windows XP and Windows Server 2003.  As with most tools, there are separate versions of TraceView that support 32-bit and native 64-bit environments.

 

Please note that TraceView does not support Windows 2000 at the present time.

 

Running The Tool

TraceView supports both command line and GUI operations.  Start TraceView as you would any other Windows-based program, and refer to the help for more details.

TraceView requires that 3 DLLs be present either in the same directory with TraceView.exe or in the execution path on the system where TraceView is being run. These DLLs are: mspdb70.dll msvcr70.dll traceprt.dll If any of these 3 DLLs are not present, TraceView will display an error when it is run, identifying the DLL that is missing. You will find mspdb70.dll and msvcr70.dll for x86 architecture systems in the \winddk\buildNumber\bin\x86 directory.  You will find traceprt.dll in the \winddk\buildNumber\tools\tracing\i386 directory.  The DLLs for IA64 systems may be found in the IA64 specific directories, similar to those above.

 

When using TraceView in GUI mode, TraceView will limit the number of trace events that are saved for display.  The default number of messages to save is 50000 per trace.  If you run TraceView on a system with limited memory, or you are simultaneously displaying multiple traces, you may need to set this maximum to a lower number.  The maximum number of trace events can be set by selecting “File”, “Set Maximum Trace Entries…” menu item from TraceView’s main window.  Note that the maximum value may only be set when no traces are currently being displayed by TraceView, and that the maximum number of trace entries applies to each trace display.

 

While TraceView may be generally useful to anyone using Event Tracing for Windows, this tool was designed specifically for Windows driver developers and testers.  Driver writers can easily make use of Windows built-in software event tracing facilities through the use of WPP Software Tracing.  WPP Software Tracing makes it particularly convenient to add trace events to a driver, and even to automatically convert DbgPrint(…) message output to event trace output.

 

WPP Software Tracing is described in the DDK documentation under the topic “WPP Software Tracing.”

 

Limitations

As a newly developed utility, TraceView does have a few limitations.  We hope these limitations do not seriously impact your use of this utility.

 

  • At this time TraceView does not support Windows 2000.
  • There is presently no way to save the output displayed by TraceView.  This includes cut and paste, which is presently not supported.  Note that before starting a trace, you can choose to have a binary trace file written to disk, which WILL save all your data.
  • It is up to the user to set TraceView’s “maximum number of trace entries” parameter to a value which will work properly on the system where TraceView is being used.  Failure to set this parameter properly can result in TraceView running out of memory and aborting or exiting without warning.
