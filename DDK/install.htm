Getting Started

The Microsoft® Windows® codename "Longhorn" Driver Kit (LDK) provides a development environment for creating drivers for Windows Longhorn, Microsoft Windows Server™ 2003, Microsoft Windows XP, and Microsoft Windows 2000. The LDK provides additional driver-specific headers, libraries, sources, tools, test frameworks, and documentation that are used to develop drivers for Windows Longhorn.

Installing the LDK

To install the Microsoft Windows Longhorn Driver Kit, you must be able to access the media from Windows, either locally or through a network.

A development machine running Microsoft Windows NT® 4.0, Microsoft Windows 95, Microsoft Windows 98, or Microsoft Windows ME is not supported when developing drivers for Windows 2000, Windows XP, Windows Server 2003, or Windows Longhorn.

Developing 16-bit drivers, such as virtual device driver (VxD) and Windows ME, is not supported on a Windows 64-bit platform.

The LDK Test Harness provides end-to-end support for automated testing of Windows-based drivers and hardware, and is directed at those who use the LDK to test, validate, and certify drivers. The Test Harness provides an automated, deployable framework for managing tests for development teams that create hardware and drivers for Windows and validate Windows configurations.

The LDK Test Harness is an initial preview that does not show full functionality. It does show the basic structure of the tool and the user flow as it will be implemented in the Longhorn timeframe.

System Requirements

The system requirements for using this LDK on a driver-development computer are:

  • Microsoft Windows Longhorn, Windows Server 2003, Windows XP, Windows XP Service Pack 1, or Windows 2000.
  • CD-ROM or DVD drive.
  • At least 128MB RAM. (256MB of RAM or more is recommended).
  • At least a 574MB hard-disk space for a full installation. Significantly more space may be required to compile all samples; up to 920MB.


The system requirements for using this LDK on a driver-testing computer are:

  • Windows Longhorn, Windows Server 2003, Windows XP, Windows XP Service Pack 1, or Windows 2000. Builds may be retail, free, or checked.
  • At least 128MB of RAM. For checked builds, 256 MB or more of RAM is recommended.
  • A second computer that can run Windows 2000, Windows XP, or Windows Server 2003 for kernel debugging of your drivers under development. If desired, this computer can be your driver-development system.


The system requirements for using this LDK on a Test Controller computer are:

  • Windows Longhorn, Windows Server 2003, or Windows XP Service Pack 1.
  • CD-ROM or DVD drive.
  • At least 128MB RAM. (256MB of RAM or more is recommended).
  • At least 59 MB of hard disk space for a full installation. Significantly more space may be required to load all test content.
  • Microsoft SQL Server 2000 Desktop Engine (MSDE). See “Microsoft SQL Server 2000 Desktop Engine (MSDE Installation)” below.
  • Microsoft .NET Framework version 1.1. (You can download and execute this Web package from the Internet).

LDK Installation

Note:    To correctly install the LDK, you must have Administrator privileges.
Note:    If a type of installation (see step 6 below) is not available, this normally indicates that all system requirements have not been met.
Note:    Installation of the LDK Test Harness Controller includes the LDK Test Harness Studio by default.
Note:    Do not install this LDK over previous versions of this LDK or any other Driver Development Kit (DDK). Improper installation of the LDK can cause the use of old and incorrect files, which leads to unreliable results in any drivers that you develop.
Note:  The installation path and icon placement have changed. The default installation path is now x:\WinDDK\Kit <BldNum>. The icons can be found at Programs\Development Kits\Windows LDK <BldNum>.

To install the LDK:

  1. From the media, run the setup.exe file.
  2. If installing on a Windows 2000 computer, click Run the program from it's current location. If installing on a Windows Longhorn, Windows Server 2003 or Windows XP computer, click Open.
  3. To install the LDK, click Next.
  4. Read and accept the Software License Agreement, and then click Next.
  5. Select the installation path (it is suggested to use the path that appears), and click Next.
  6. Choose the type of installation that you want, and then click Next:
    • Development System installs development tools and content.
    • Test Server installs LDK Test Harness Controller.
    • System under Test installs the LDK Test Harness Client.
    Note:    To select the System Under Test check box, you might have to clear the Development System check box.
  7. Select the components that you want to install, and then click Next.
  8. If you selected Development System in step 6, select one of the following under Test Harness Controller Information, and then click Next:
    • If the Test Harness Controller is the local computer, select the Local computer check box.
    • If the Test Harness Controller is on another computer, select the Remote computer check box and type the computer name.
    • If you selected Test Server in step 6, type the following Test Server Information in the text boxes, and then click Next:
    • Account name: Your domain or local user name.
    • Password: Your domain or local password that is associated with the user name above.
    • Domain: Your domain name. If your computer is on a workgroup, you must type the computer name.
    • Database name: Any user-defined name for the database (such as MYDATABASE).
    • SA Password: The sapwd (such as "<a secure password>") that you supplied during MSDE setup.
    • If you selected System under Test in step 6, select one of the following under Test Harness Controller Information, and then click Next:
    • If the Test Harness Controller is the local computer, select Local computer.
    • If the Test Harness Controller is on another computer, select Remote computer and type the computer name in the text box.
  9. To confirm that you want to install the LDK, verify that the installation path is correct.
  10. To install the LDK, click Next.
  11. After the installation is complete, click Finish.
Note:    Before installing the LDK Test Harness, you must download and install the most current version of MSDE to your computer. Because MSDE provides the required datastore for the Test Harness, you cannot install the Test Harness component of the LDK unless you install MSDE first. Currently, you cannot install MSDE and the LDK Test Harness on a computer that is running SQL Server.

Microsoft SQL Server 2000 Desktop Engine (MSDE Installation)

  1. From the computer where you want to install the Test Controller, download and execute the MSDE downloadable web package http://download.microsoft.com/download/8/7/5/875e38ea-e582-4ee2-9485-b459cd9c0082/sql2kdesksp3.exe file from the Internet.
  2. To proceed, click Open.
  3. In response to Do you want to Run or Save sql2kdesksp3.exe from download.microsoft.com?, click Run.
  4. Read and accept the Software License Agreement.
  5. Select the default installation path.
  6. To extract the MSDE setup, click Finish, and wait for the extraction to complete.
  7. On the taskbar, Click Start and then click Run. In the Run dialog box, type the path to which you saved the MSDE setup, followed by:

    \msde\setup securitymode=SQL disablenetworkprotocols=0 sapwd="<a secure password>" instancename=MSLDK

    Note: Replace "<a secure password>" with your secure password. When installing the Test Controller, you will need to supply this password.

  8. To launch the MSDE installation, press ENTER and wait for the installation to complete.

LDK Test Harness Initialization

NOTE: Following installation of the LDK Test Harness Controller, Studio, and one or more Clients, perform the following steps to enable the user to schedule jobs against the client test systems.

  1. Click Start, point to Programs, and click LDK Test Harness Studio.
  2. On the LDK Test Harness Studio menu, point to Asset, then click My Assets. The Default Machine Pool node and the System node should be visible.
  3. Select the Default Machine Pool node.
  4. To list the Controller and Client systems, click the Search button (Green Triangle). These appear in the Default Machine Pool node after the Sysparse inventory is complete on each system, which may take up to 10 minutes to run following either Controller Setup or Client Setup.
  5. On the Machine list tab, select the machine name of the Controller and drag it to the System node.
  6. After the Controller is in the System node, right-click on the machine name and click Edit Machine Details. In the Machine Details dialog box, add machine information to the required text boxes (those with the label in bold type).
    • Change the Machine Status from Manual to Ready
    • From the Vendor Name drop-down list, select an applicable vendor name,
    • From the Vendor Division drop-down list, select an applicable vendor division.
    • From the Model Name drop-down list, select an applicable model name.
    • From the Series drop-down list, select an applicable series.
    • In the Serial Number text box, type a serial number.
    • In the Asset Tag text box, type any six-digit number.
    • From the Machine Status drop-down list, select Ready.
  7. To save the machine details, click Save.
  8. Right-click the $ node and select Add Machine Pool.
  9. Create a new machine pool, and select the Controller machine name as the job delivery agent.
  10. To list the Client systems, select the Default Machine Pool node, and click the Search button.
  11. From the Default Machine Pool, select each of the Client systems and drag them to the newly created Machine Pool.
  12. To list the Client systems, select the new Machine Pool node, and click the Search button.
  13. Right-click each Client machine name and Edit Machine Details. For each Client system, set the Machine Status to Ready.
  14. The user is now able to schedule and run jobs on the Client machines.

Note: On systems with large hard disk sectors such as "FAT 16" file systems, the Setup program may miscalculate the amount of space that is needed to attempt an installation even though the space is unavailable.
Note: You can move multiple machines simultaneously by holding down CTRL, selecting the machines to be moved, and then right-clicking them to select the Move Asset(s) window.

LDK Test Harness Job Content Access

NOTE: Following installation of the LDK Test Harness Controller and Studio, perform the following steps to enable the user to access the Test Harness Job Content provided as part of the LDK Test Harness.

  1. In the Test Studio, click on the Admin menu item, then click on the Parameters menu item to launch the WTT Parameters dialog.
  2. Select your Test Controller in the Show Parameters from the controller: drop-down list box.
  3. Click on the TestRoot parameter, then click the Edit button.
  4. In the Value field, type \\"Your Test Controller Name"\LDKTests.

    Note: the text "Your Test Controller Name" should be replaced by the name of the PC where you installed the test controller. For example, if you installed the Test Controller on a computer named MyController, you would type \\MyController\LDKTests.

  5. Click on the OK button to close the Edit Parameter window.
  6. Click on the Close button to close the WTT Parameters window.

Installing the Debuggers

The Debugging Tools for Windows package features WinDbg, a powerful debugger with a graphical interface that can debug both user-mode and kernel-mode code. This package also includes the debuggers KD, NTSD, and CDB, and many additional tools. The documentation in this package describes the use of these debuggers and includes tips for user-mode and kernel-mode debugging.

Debugging Tools for Windows is available in 32-bit and 64-bit versions. The 32-bit version is appropriate for most users. However, if you want to debug an application on a processor that is based on 64 bits, you should select the 64-bit version. The 64-bit version can be installed only on 64-bit versions of Windows.

These debugging tools require approximately 21 MB of hard disk space.

To obtain the most current version of Debugging Tools for Windows, see  Microsoft Debugging Tools.

If you cannot install Debugging Tools for Windows from this Internet site, then use one of the following links:

  1. Install 32-bit Debugging Tools for Windows (Required for most customers)
  2. Install 64-bit Debugging Tools for Windows

Verifying Longhorn Driver Kit Installation

To verify that you have properly installed the LDK, perform the following steps:

  1. Open a command window with environment variables set for a Windows Longhorn checked build environment, using the Programs/Development Kits/Windows LDK <BldNum>/Build Environments/Windows Longhorn/Windows Longhorn IA-32 Checked Build Environment menu item.
  2. In the command window, type the command build -cZ. This command compiles and links the complete set of installed sources. This can take more than 30 minutes to complete, depending on which LDK components are installed and the system on which the build is being performed.
  3. The build command should complete with no errors and no warnings displayed.
